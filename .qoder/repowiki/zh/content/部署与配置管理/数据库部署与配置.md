## 引言
本项目基于 Flask 框架，使用 SQLAlchemy 作为 ORM 工具，通过 PyMySQL 驱动连接 MySQL 数据库。在生产环境中，数据库的稳定性和安全性至关重要。本文档旨在提供完整的 MySQL 数据库部署建议，涵盖从服务器初始化到监控的各个环节，确保系统在高并发场景下的可靠运行。

## 数据库服务器安装与初始化
在生产环境中，建议使用稳定的 MySQL 8.0 或 MariaDB 10.6+ 版本。安装完成后，应执行以下初始化步骤：

1. 运行 `mysql_secure_installation` 脚本，设置强密码、移除匿名用户、禁用远程 root 登录。
2. 创建专用数据库 `glzx_xmt`，用于存储本项目数据。
3. 配置 `my.cnf` 文件，优化 InnoDB 引擎参数，如 `innodb_buffer_pool_size` 设置为物理内存的 70%-80%。
4. 启用二进制日志（binlog）以支持数据恢复和主从复制。

## 字符集与排序规则设置
为确保中文内容正确存储与检索，必须统一设置字符集。在 `my.cnf` 中配置：

```ini
[client]
default-character-set = utf8mb4

[mysql]
default-character-set = utf8mb4

[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
```

在创建数据库时显式指定：

```sql
CREATE DATABASE glzx_xmt CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

应用代码中已通过连接字符串 `?charset=utf8mb4` 明确指定字符集，确保与数据库一致。

## 连接池配置
SQLAlchemy 使用连接池管理数据库连接。虽然当前代码未显式配置池参数，但在生产环境中应通过环境变量进行优化：

- `SQLALCHEMY_POOL_SIZE`: 建议设置为 20-50，根据并发用户数调整。
- `SQLALCHEMY_MAX_OVERFLOW`: 设置为 `POOL_SIZE` 的 1-2 倍，应对突发流量。
- `SQLALCHEMY_POOL_TIMEOUT`: 建议 30 秒，避免请求长时间挂起。
- `SQLALCHEMY_POOL_RECYCLE`: 设置为 3600 秒（1小时），防止连接因超时被数据库关闭。

这些参数可通过 Flask 配置注入，例如：

```python
app.config['SQLALCHEMY_ENGINE_OPTIONS'] = {
    'pool_size': int(os.environ.get('SQLALCHEMY_POOL_SIZE', 20)),
    'max_overflow': int(os.environ.get('SQLALCHEMY_MAX_OVERFLOW', 40)),
    'pool_timeout': int(os.environ.get('SQLALCHEMY_POOL_TIMEOUT', 30)),
    'pool_recycle': int(os.environ.get('SQLALCHEMY_POOL_RECYCLE', 3600))
}
```

## 用户权限分配
为最小化安全风险，应创建专用数据库用户并授予最小必要权限：

```sql
-- 创建生产用户
CREATE USER 'glzx_xmt_user'@'web-server-ip' IDENTIFIED BY '强密码';

-- 授予数据库权限
GRANT SELECT, INSERT, UPDATE, DELETE ON glzx_xmt.* TO 'glzx_xmt_user'@'web-server-ip';

-- 刷新权限
FLUSH PRIVILEGES;
```

禁止使用 root 账户连接生产数据库。开发环境可适当放宽权限，但生产环境必须严格限制。

## 远程访问安全策略
生产数据库不应直接暴露于公网。推荐架构：

1. 数据库服务器置于内网，仅允许应用服务器通过私有 IP 访问。
2. 使用防火墙（如 iptables）限制 3306 端口的访问来源。
3. 启用 SSL 加密连接，防止数据在传输中被窃听。
4. 定期审计数据库日志，监控异常登录行为。

应用通过环境变量 `DB_HOST` 指定数据库地址，便于在不同环境中切换内网或本地连接。

## 多环境数据库配置切换
项目通过环境变量实现多环境配置分离：

- **开发环境**：使用本地 SQLite 或测试 MySQL，`DATABASE_URL` 可为空，使用默认值。
- **测试环境**：使用独立测试数据库，通过 CI/CD 注入 `DB_HOST`、`DB_USER` 等变量。
- **生产环境**：通过部署平台（如 Docker、Kubernetes）注入生产数据库连接信息。

配置优先级：`DATABASE_URL` > 分段环境变量（`DB_USER`, `DB_PASSWORD` 等）。此设计便于在不同环境中灵活切换。

## 数据库备份与恢复
### 备份脚本示例（每日全量备份）
```bash
#!/bin/bash
BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/glzx_xmt_$DATE.sql"

# 执行备份
mysqldump -u backup_user -p'backup_password' --single-transaction --routines --triggers glzx_xmt > $BACKUP_FILE

# 压缩并删除7天前的备份
gzip $BACKUP_FILE
find $BACKUP_DIR -name "glzx_xmt_*.sql.gz" -mtime +7 -delete
```

### 恢复脚本示例
```bash
#!/bin/bash
BACKUP_FILE="$1"
mysql -u root -p glzx_xmt < $BACKUP_FILE
```

建议将备份文件存储于独立存储设备或云存储，并定期验证恢复流程。

## 数据库监控脚本示例
### 连接数监控
```bash
#!/bin/bash
MAX_CONNECTIONS=100
CURRENT=$(mysql -u monitor -p'monitor_password' -e "SHOW STATUS LIKE 'Threads_connected';" | awk 'NR==2 {print $2}')

if [ $CURRENT -gt $MAX_CONNECTIONS ]; then
    echo "警告：数据库连接数过高: $CURRENT" | mail -s "MySQL 警告" admin@example.com
fi
```
