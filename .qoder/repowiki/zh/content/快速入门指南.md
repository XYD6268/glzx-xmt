## 简介

本指南旨在帮助新手开发者快速搭建 **glzx-xmt 摄影比赛投票系统** 的本地开发环境。该系统基于 Flask 构建，支持用户注册、照片上传、管理员审核、投票等功能，并具备三级权限管理体系。

通过本指南，您将能够在15分钟内完成环境部署，成功访问登录页面并上传测试照片，验证系统基本功能正常运行。

## 克隆仓库

首先，您需要将项目代码从远程仓库克隆到本地。

### 操作步骤

1. 打开终端（Windows：CMD/PowerShell；macOS/Linux：Terminal）
2. 执行以下命令（请替换为实际的仓库地址）：

```bash
git clone https://github.com/your-username/glzx-xmt.git
cd glzx-xmt
```

> **说明**：如果尚未安装 Git，请先前往 [https://git-scm.com](https://git-scm.com) 下载并安装。

## 安装Python依赖

项目依赖通过 `pyproject.toml` 和 `requirements.txt` 提供。推荐使用虚拟环境以避免依赖冲突。

### 推荐方式：使用 pip 和虚拟环境

#### 1. 创建并激活虚拟环境

```bash
# 创建虚拟环境
python -m venv venv

# Windows 激活
venv\Scripts\activate

# macOS/Linux 激活
source venv/bin/activate
```

#### 2. 安装依赖

```bash
pip install -r requirements.txt
```

> **说明**：`requirements.txt` 文件未直接提供，但 `README_DEPENDENCIES.md` 中列出了所有依赖项。您也可以根据 `pyproject.toml` 中的 `dependencies` 字段手动创建。

#### 手动创建 requirements.txt（可选）

根据 `pyproject.toml` 和 `README_DEPENDENCIES.md`，创建 `requirements.txt` 文件内容如下：

```
Flask==2.3.3
Flask-SQLAlchemy==3.0.5
Pillow==10.0.1
PyMySQL==1.1.0
Werkzeug==2.3.7
pandas
openpyxl
```

然后运行 `pip install -r requirements.txt`。

### 替代方式：使用 Poetry（高级用户）

若使用 Poetry 管理依赖：

```bash
poetry install
```

> **注意**：`pyproject.toml` 中已声明 `build-backend = "hatchling"`，因此默认不使用 Poetry。如需使用，请修改 `pyproject.toml` 或手动安装依赖。

## 配置数据库连接

项目支持 **SQLite（测试用）** 和 **MySQL（生产用）** 两种数据库。

### 方式一：使用 SQLite（推荐新手）

无需额外配置，直接使用 `app_test.py` 即可。

- 数据库文件：`photos.db`（自动生成）
- 优点：零配置、轻量、适合本地测试

### 方式二：使用 MySQL（需手动配置）

1. 确保已安装并运行 MySQL 服务
2. 创建数据库（如 `glzx_xmt`）
3. 修改 `src/app.py` 中的数据库连接配置：

```python
app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql+pymysql://用户名:密码@localhost/数据库名'
```

> **环境变量支持**：项目支持通过环境变量配置数据库，优先级更高：
> - `DATABASE_URL`：完整数据库连接字符串
> - `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`, `DB_NAME`：分段配置

## 初始化数据表结构

数据库表结构通过 SQLAlchemy ORM 定义，需手动创建数据表。

### 对于 SQLite（app_test.py）

在 `src/app_test.py` 的末尾添加初始化代码（若不存在）：

```python
if __name__ == '__main__':
    with app.app_context():
        db.create_all()  # 创建所有表
    app.run(debug=True)
```

### 对于 MySQL（app.py）

同样，在 `src/app.py` 中确保有以下代码：

```python
if __name__ == '__main__':
    with app.app_context():
        db.create_all()
    app.run(debug=True)
```

### 执行初始化

运行以下命令创建数据表：

```bash
python src/app_test.py
```

> **说明**：首次运行会自动创建 `photos.db` 和所有数据表（如 `user`, `photo`, `vote` 等）。

## 最小化可运行示例

以下步骤验证系统是否成功运行：

### 1. 访问登录页面

- 地址：[http://127.0.0.1:5000/login](http://127.0.0.1:5000/login)
- 页面应正常加载，显示登录表单

### 2. 使用默认管理员账号登录

- 账号：`admin`
- 密码：`admin123`

> **注意**：首次运行时，管理员账号需手动创建或通过数据库初始化脚本添加。若无法登录，请检查数据库中 `user` 表是否存在管理员记录。

### 3. 上传一张测试照片

1. 登录后点击“上传照片”
2. 选择一张本地图片（支持多选）
3. 填写作品名称（可选）
4. 点击“上传”

### 4. 验证上传结果

- 查看“我的照片”页面，确认照片状态为“待审核”
- 检查 `static/uploads/` 和 `static/thumbs/` 目录，确认原图和缩略图已生成

### 5. 管理员审核照片

1. 使用管理员账号登录
2. 进入“管理面板”
3. 找到待审核照片，点击“通过”
4. 返回首页，确认照片已公开显示

至此，您已完成最小化可运行示例的全部验证。

## 常见问题与解决方案

### 1. 依赖冲突或安装失败

**问题**：`pip install` 报错，如 `Could not find a version`

**解决方案**：
- 确保使用虚拟环境
- 升级 pip：`python -m pip install --upgrade pip`
- 检查网络或使用国内镜像源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 2. 端口占用（Address already in use）

**问题**：启动时报错 `OSError: [Errno 98] Address already in use`

**解决方案**：
- 更换端口：

```python
app.run(debug=True, port=5001)
```

- 或终止占用进程：

```bash
# Linux/macOS
lsof -i :5000
kill -9 <PID>

# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F
```

### 3. 数据库连接失败（MySQL）

**问题**：`sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError) (2003, "Can't connect to MySQL server")`

**解决方案**：
- 确认 MySQL 服务已启动
- 检查用户名、密码、主机、端口是否正确
- 确认数据库已创建
- 检查防火墙设置

### 4. 数据表未创建

**问题**：运行时报错 `no such table: user`

**解决方案**：
- 确保执行了 `db.create_all()`
- 检查数据库文件路径权限
- 删除旧数据库文件（如 `photos.db`）后重新运行初始化

### 5. 无法登录（默认账号无效）

**问题**：使用 `admin/admin123` 无法登录

**解决方案**：
- 首次运行需手动创建管理员账号，或通过 SQL 插入：

```sql
INSERT INTO user (real_name, password_hash, role, class_name, qq_number) 
VALUES ('admin', 'pbkdf2:sha256:260000...（使用 generate_password_hash 生成）', 3, '管理员', '10000');
```

- 或在注册时创建账号，再通过数据库修改 `role=3`
