```mermaid
sequenceDiagram
participant 用户
participant upload_html
participant Flask
participant app_py
participant 文件系统
用户->>upload_html : 访问/upload页面

用户->>upload_html : 选择图片并填写作品名称
upload_html->>Flask : POST /upload, enctype='multipart/form-data'
Flask->>app_py : 解析多部分表单数据
app_py->>app_py : 验证上传权限
loop 处理每张照片
app_py->>app_py : 生成唯一文件名
app_py->>文件系统 : 保存原始文件到UPLOAD_FOLDER
app_py->>app_py : 生成缩略图并保存
app_py->>app_py : 创建Photo数据库记录待审核
end
app_py->>用户 : 重定向至我的照片页面
```

## 管理员审核流程

管理员通过 `/review_photo` 路由访问 `admin_review.html` 页面，对照片进行审核操作。

```mermaid
sequenceDiagram
participant 管理员
participant admin_review_html
participant Flask
participant app_py
管理员->>admin_review_html : 访问/admin/review
admin_review_html->>Flask : GET /admin/review
Flask->>app_py : 查询status=0的照片
app_py->>admin_review_html : 返回待审核照片列表
admin_review_html->>管理员 : 显示照片网格与操作按钮
管理员->>admin_review_html : 点击"通过"按钮
admin_review_html->>Flask : GET /approve_photo/<photo_id>
Flask->>app_py : 更新Photo.status为1
app_py->>管理员 : 重定向并显示成功消息
```

## 文件存储安全与防重复机制

系统通过多种机制确保文件存储安全并防止重复上传。

```mermaid
flowchart TB
A[文件上传] --> B[secure_filename处理]
B --> C[时间戳+计数器生成唯一文件名]
C --> D[保存到UPLOAD_FOLDER]
D --> E[权限控制]
E --> F{用户角色}
F --> |管理员| G[可访问所有文件]
F --> |普通用户| H[仅可访问自己文件]
H --> I[通过Photo.user_id验证]
I --> J[secure_uploaded_file路由]
J --> K[文件路径安全检查]
K --> L[防止路径遍历攻击]
M[防重复机制] --> N[数据库Photo记录]
N --> O[通过user_id+filename+时间戳确保唯一性]
O --> P[不允许多次上传同名文件]
```
