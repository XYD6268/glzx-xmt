## 引言
本项目实现了一套完整的安全防护体系，涵盖身份认证、权限控制、风控策略、数据安全等多个层面。系统通过装饰器模式实现多级权限控制，结合IP封禁、白名单、频率限制等手段有效防止恶意行为。同时，系统在密码存储、会话管理、文件访问等方面采取了多项安全措施，确保系统稳定可靠运行。

## 身份验证与权限分级控制
系统采用基于装饰器的权限控制机制，实现了三级权限体系：普通用户、管理员、系统管理员。通过`login_required`、`admin_required`和`super_admin_required`三个装饰器实现不同级别的访问控制。

```mermaid
classDiagram
class login_required {
+检查用户登录状态
+验证账户活跃状态
}
class admin_required {
+继承login_required功能
+验证角色等级≥2
}
class super_admin_required {
+继承login_required功能
+验证角色等级≥3
}
admin_required --> login_required : "继承"
super_admin_required --> login_required : "继承"
```

## 登录频率限制与IP封禁机制
系统实现了完善的登录频率限制和IP封禁机制，通过`IpBanRecord`模型记录封禁信息，并在关键入口进行IP检查。

```mermaid
sequenceDiagram
participant 用户
participant 登录接口
participant IP检查
participant 登录频率检查
participant 数据库
用户->>登录接口 : 提交登录请求
登录接口->>IP检查 : 获取客户端IP
IP检查->>数据库 : 查询IpBanRecord
数据库-->>IP检查 : 返回封禁状态
alt IP被封禁
IP检查-->>登录接口 : 返回封禁信息
登录接口-->>用户 : 显示封禁提示
else IP未被封禁
登录接口->>登录频率检查 : 检查登录频率
登录频率检查->>数据库 : 查询LoginRecord
数据库-->>登录频率检查 : 返回登录记录
alt 超过频率限制
登录频率检查-->>登录接口 : 返回超限信息
登录接口->>数据库 : 封禁IP和关联用户
数据库-->>登录接口 : 确认封禁
登录接口-->>用户 : 显示封禁提示
else 未超限
登录接口->>数据库 : 记录登录信息
数据库-->>登录接口 : 确认记录
登录接口-->>用户 : 登录成功
end
end
```

## 白名单机制
系统提供了IP白名单和用户白名单两种机制，允许受信任的IP或用户免受风控限制。通过`IpWhitelist`和`UserWhitelist`模型实现，并在风控检查时优先判断白名单状态。

```mermaid
flowchart TD
Start([开始]) --> CheckWhitelist["检查白名单"]
CheckWhitelist --> IsInWhitelist{"是否在白名单?"}
IsInWhitelist --> |是| Bypass["绕过风控检查"]
IsInWhitelist --> |否| NormalCheck["执行正常风控检查"]
Bypass --> End([结束])
NormalCheck --> End
```

```mermaid
sequenceDiagram
```

## 密码存储与会话安全
系统采用Werkzeug的安全函数进行密码哈希处理，使用`generate_password_hash`和`check_password_hash`确保密码存储安全。会话管理通过Flask的session机制实现，结合SECRET_KEY进行加密。

```mermaid
classDiagram
class PasswordSecurity {
+generate_password_hash()
+check_password_hash()
+使用PBKDF2-SHA256算法
+自动加盐
}
class SessionSecurity {
+基于SECRET_KEY加密
+存储在客户端cookie
+包含用户ID、角色等信息
}
PasswordSecurity --> User : "用于密码存储"
SessionSecurity --> Login : "用于会话管理"
```

